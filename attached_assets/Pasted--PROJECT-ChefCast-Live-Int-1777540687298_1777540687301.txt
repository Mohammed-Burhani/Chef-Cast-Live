## DATABASE SCHEMA — GENERATE ALL MIGRATIONS
=====================================================================

Generate the following PostgreSQL tables with Supabase RLS enabled:

TABLE: profiles
- id uuid (FK auth.users, PRIMARY KEY)
- username text UNIQUE NOT NULL
- display_name text
- avatar_url text
- xp_total integer DEFAULT 0
- current_streak integer DEFAULT 0
- longest_streak integer DEFAULT 0
- last_cooked_at timestamptz
- subscription_tier text DEFAULT 'free' -- 'free' | 'premium'
- created_at timestamptz DEFAULT now()

TABLE: episodes
- id uuid PRIMARY KEY
- title text NOT NULL
- description text
- thumbnail_url text
- video_url text
- broadcast_at timestamptz
- is_live boolean DEFAULT false
- chef_name text
- season integer
- episode_number integer
- created_at timestamptz DEFAULT now()

TABLE: recipes
- id uuid PRIMARY KEY
- episode_id uuid FK episodes
- title text NOT NULL
- description text
- difficulty text -- 'easy' | 'medium' | 'hard'
- prep_time_minutes integer
- cook_time_minutes integer
- servings integer DEFAULT 4
- created_at timestamptz DEFAULT now()

TABLE: ingredients
- id uuid PRIMARY KEY
- recipe_id uuid FK recipes
- name text NOT NULL
- quantity text
- unit text
- is_optional boolean DEFAULT false
- substitutes jsonb -- array of substitute options
- step_number integer -- which step this ingredient is first used

TABLE: recipe_steps
- id uuid PRIMARY KEY
- recipe_id uuid FK recipes
- step_number integer NOT NULL
- instruction text NOT NULL
- duration_seconds integer -- auto-timer trigger
- technique_tag text -- e.g. 'knife_work', 'sauce', 'plating'
- tip text -- optional chef tip

TABLE: polls
- id uuid PRIMARY KEY
- episode_id uuid FK episodes
- question text NOT NULL
- options jsonb -- [{id, label, votes}]
- is_active boolean DEFAULT false
- winner_option_id text -- populated when poll closes
- created_at timestamptz DEFAULT now()
- closes_at timestamptz

TABLE: poll_votes
- id uuid PRIMARY KEY
- poll_id uuid FK polls
- user_id uuid FK profiles
- option_id text NOT NULL
- voted_at timestamptz DEFAULT now()
- UNIQUE(poll_id, user_id)

TABLE: badges
- id uuid PRIMARY KEY
- slug text UNIQUE NOT NULL
- name text NOT NULL
- description text
- icon_url text
- xp_reward integer DEFAULT 50
- unlock_criteria jsonb -- conditions object for server-side check

TABLE: user_badges
- id uuid PRIMARY KEY
- user_id uuid FK profiles
- badge_id uuid FK badges
- earned_at timestamptz DEFAULT now()
- UNIQUE(user_id, badge_id)

TABLE: user_cook_sessions
- id uuid PRIMARY KEY
- user_id uuid FK profiles
- episode_id uuid FK episodes
- recipe_id uuid FK recipes
- completed_steps integer[] DEFAULT '{}'
- completed boolean DEFAULT false
- dish_photo_url text
- started_at timestamptz DEFAULT now()
- completed_at timestamptz
- score integer DEFAULT 0

TABLE: community_posts
- id uuid PRIMARY KEY
- user_id uuid FK profiles
- episode_id uuid FK episodes
- recipe_id uuid FK recipes
- photo_url text NOT NULL
- caption text
- likes integer DEFAULT 0
- created_at timestamptz DEFAULT now()

TABLE: qa_questions
- id uuid PRIMARY KEY
- user_id uuid FK profiles
- episode_id uuid FK episodes
- question text NOT NULL
- is_answered boolean DEFAULT false
- is_selected_for_broadcast boolean DEFAULT false
- answer text
- upvotes integer DEFAULT 0
- created_at timestamptz DEFAULT now()

TABLE: mystery_box_submissions
- id uuid PRIMARY KEY
- user_id uuid FK profiles
- episode_id uuid FK episodes
- mystery_ingredient text NOT NULL
- dish_idea text NOT NULL
- is_featured_on_tv boolean DEFAULT false
- submitted_at timestamptz DEFAULT now()

TABLE: leaderboard_scores
- id uuid PRIMARY KEY
- user_id uuid FK profiles
- episode_id uuid FK episodes
- score integer DEFAULT 0
- rank integer
- breakdown jsonb -- {speed, accuracy, engagement, streak_bonus}
- calculated_at timestamptz DEFAULT now()

RLS POLICIES TO GENERATE:
- profiles: users can read all, update only their own
- poll_votes: users can insert once per poll, read all
- user_badges: users can read all, only edge functions can insert
- community_posts: authenticated users can insert, all can read
- user_cook_sessions: users can CRUD only their own
- qa_questions: authenticated can insert, all can read
- mystery_box_submissions: authenticated can insert, all can read

Enable Realtime on: polls, poll_votes, episodes, qa_questions

## GAMIFICATION LOGIC — GENERATE FULL IMPLEMENTATION
=====================================================================

XP EVENTS (define in constants/gamification.ts):
- COOK_SESSION_COMPLETED = 100 XP
- POLL_VOTED = 10 XP
- STEP_COMPLETED = 5 XP
- DISH_PHOTO_UPLOADED = 25 XP
- QA_QUESTION_SUBMITTED = 15 XP
- MYSTERY_BOX_SUBMITTED = 50 XP
- MYSTERY_BOX_FEATURED = 200 XP
- STREAK_3_DAYS = 75 XP bonus
- STREAK_7_DAYS = 200 XP bonus
- STREAK_30_DAYS = 1000 XP bonus
- BADGE_EARNED = badge.xp_reward XP

BADGE UNLOCK CRITERIA (edge function: award-badge/index.ts):
- knife_master: complete any 5 steps tagged 'knife_work'
- sauce_expert: complete any 5 steps tagged 'sauce'
- speed_demon: complete a cook session in under 80% of estimated time
- streak_3 / streak_7 / streak_30: cook N days in a row
- mystery_winner: have a mystery box submission featured on TV
- community_star: receive 50 total likes on dish photos
- poll_voter: vote in 10 polls
- first_cook: complete first cook session
- perfect_plate: upload a photo AND complete all steps

STREAK LOGIC (hooks/useStreak.ts):
- Streak increments when user completes a cook session on a new calendar day
- Streak resets to 0 if no session completed in last 48 hours
- Streak freeze items (premium): prevent reset once per 7 days
- Read/write streak to both MMKV (offline) and Supabase profiles (sync)

SCORE CALCULATION for leaderboard:
  base_score = steps_completed * 10
  speed_bonus = max(0, (expected_duration - actual_duration) / 60) * 5
  accuracy_bonus = poll_votes_cast * 10
  engagement_bonus = (questions_asked + photo_uploaded ? 25 : 0)
  streak_multiplier = 1 + (current_streak * 0.05) — cap at 2.0
  final_score = (base_score + speed_bonus + accuracy_bonus + engagement_bonus)
                * streak_multiplier

## INGREDIENT SCANNER — SUPABASE EDGE FUNCTION
=====================================================================

Generate supabase/functions/ingredient-detection/index.ts:

1. Receive base64 image from client
2. Call Anthropic API (claude-haiku-4-5 model) with vision:
   Prompt: "Analyze this image of a refrigerator or kitchen counter.
   List all visible food ingredients as a JSON array of objects:
   [{name: string, quantity_estimate: string, confidence: 'high'|'medium'|'low'}]
   Return ONLY valid JSON, no markdown."
3. Parse response, return ingredients array
